# Skill Fixtures

Use this directory for deterministic test fixtures, sample provider responses, mock inputs, and expected structured outputs.

Fixtures should allow tests to run without uncontrolled external state where practical.

Examples:

```text
provider_success.json
provider_timeout.json
memory_empty.json
memory_with_default_location.json
expected_current_weather_response.json
expected_awaiting_input_state.json
```

Do not include sensitive user data in fixtures.
