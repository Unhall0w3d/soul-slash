# Soul/ Branding

Soul/ uses a **techno-grimoire** visual direction: arcane structure, necromantic atmosphere, luminous interface glyphs, and local AI orchestration motifs.

This branding is for a project repository, not a commercial product. It should feel like a local intelligence system being carefully built, tested, constrained, and taught.

## Project names

Use these consistently:

- **Soul/** for the project identity
- **soul-slash** for the repository name
- **Soul Slash** as a readable alias

## Brand statement

Soul/ is a local intelligence substrate that combines language models, deterministic skills, safety gates, recoverable workflows, and human-approved memory.

## Guiding idea

The model is the language organ. Soul/ is the operating layer around it.

## Visual direction

The selected direction mixes:

- open grimoires / codices
- sigils, circles, wards, and geometric diagrams
- spectral violet light
- cyan circuitry and interface traces
- ember-gold accents
- dark parchment / black glass / necromantic slate backgrounds
- AI/runtime symbols, verification marks, memory nodes, and workflow loops

The design should look arcane and technical without becoming fantasy cosplay nonsense. The project is about safe local automation, not summoning a daemon because somebody forgot to escape a shell argument.

## Core messages

Use language connected to the actual long-term project:

- local-first intelligence substrate
- verified actions
- recoverable workflows
- deterministic skills
- safety gates
- human-approved memory
- workflow orchestration
- reflection and rule promotion
- model as language organ
- operating layer around the model

Avoid generic product marketing like:

- “revolutionary”
- “next-generation platform”
- “AI-powered productivity”
- “enterprise-ready”
- “seamless experience”

Humanity has suffered enough.

| Necro Slate | `#151922` | panels, cards, quiet surfaces |

| Spectral Teal | `#00E2D6` | AI/circuit/data accent |
| Ember Gold | `#FFB14A` | warmth, warning, recovery |
| Pale Silver | `#E6ECF1` | primary text and logo metal |

## Typography direction

Generated mockups use a serif display face for the wordmark and clean sans/mono pairings for documentation/UI.

Suggested real-world equivalents:

- Display: Cinzel, Cormorant Garamond, or similar
- Body/UI: Inter, Source Sans 3, or similar
- Code/UI mono: JetBrains Mono, IBM Plex Mono, or similar
