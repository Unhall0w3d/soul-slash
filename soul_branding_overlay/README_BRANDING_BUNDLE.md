# Soul/ Branding Bundle Overlay

This overlay adds a first-pass branding bundle for the **Soul/** project and the **soul-slash** repository.

## Intent

The visual direction mixes:

- arcane / magical / necromantic symbolism
- local intelligence / AI / systems orchestration themes
- a dark, technical, project-oriented tone

This bundle is meant for a **project repository**, not a commercial product page.

## Included Assets

- `assets/brand/soul-slash-brand-board.png`
  - Full brand board / reference sheet.
- `assets/brand/soul-slash-repo-header.png`
  - Wide repository header / README hero image.
- `assets/brand/soul-slash-supporting-scene.png`
  - Supporting illustration for docs, wiki pages, or future presentations.
- `assets/brand/soul-slash-primary-mark.png`
  - Primary emblem / project mark.
- `assets/brand/soul-slash-repo-icon.png`
  - Square repo/app icon.

## Suggested Usage

- Use `soul-slash-repo-header.png` near the top of `README.md`.
- Use `soul-slash-repo-icon.png` as the repository social preview seed or as the base for downstream square icon assets.
- Use `soul-slash-brand-board.png` as the visual reference for future repo graphics.
- Use `soul-slash-primary-mark.png` anywhere a standalone mark is needed.
- Use `soul-slash-supporting-scene.png` in docs where a thematic visual is useful.

## Notes

- This overlay adds only branding assets and documentation.
- It does not modify repository settings automatically.
- If you want, the next step can be a second overlay that wires the header into `README.md` and adds a `docs/BRANDING.md` section reference from the main project README.
